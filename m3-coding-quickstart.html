Quick Start Guide

AI Coding Setup M3 / 16GB

Everything you need to turn a MacBook M3 (16GB) into an AI-powered coding machine. No audio, video, or image generation — just coding tools, local models, and Claude Code.

Section 01

Your Hardware Reality

The M3 with 16GB unified memory is a capable coding machine, but you need to be strategic about which local models you run. Here's what you're working with.

What You Have

  • Apple M3 chip — ~100GB/s memory bandwidth
  • 16GB unified memory (shared between CPU, GPU, and models)
  • Hardware-accelerated ML via Metal / Neural Engine
  • Enough for 7B–14B models comfortably

The Constraint

  • macOS + apps use ~5–7GB, leaving ~9–11GB for models
  • Max comfortable model size: 14B parameters (Q4 quantised)
  • 32B+ models will swap to disk — usable but slow
  • Run one model at a time for best performance

The Strategy

  • Cloud-first: Claude Code for complex tasks (best quality)
  • Local for: quick questions, offline work, private data
  • Pick 2–3 local models max, not the full roster
  • Don't try to replicate a 64GB workstation — lean into the hybrid
Section 02

Installation Order

Order matters — later tools depend on earlier ones. Copy and paste these blocks in sequence.

Step 1 — Package Manager & Core Tools
# Homebrew (package manager for macOS)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc

# Core dev tools
brew install git python@3.12 node nvm uv pipx wget jq
Step 2 — Terminal & Container
# Better terminal
brew install --cask iterm2

# Docker (for containerised projects)
brew install --cask docker
Step 3 — AI Tools
# Ollama (local LLM inference)
brew install ollama

# Claude Code (cloud AI coding assistant)
npm install -g @anthropic-ai/claude-code
Step 4 — Python Environment
# uv is 10-100x faster than pip for package management
pip install uv --break-system-packages

# For each project:
cd ~/Projects/my-project
uv venv
source .venv/bin/activate
uv pip install -r requirements.txt
Step 5 — IDE
# VS Code (recommended) or Cursor
brew install --cask visual-studio-code

# Essential extensions (install from VS Code):
# - Claude (Anthropic) — AI coding in the IDE
# - Python (Microsoft) + Pylance
# - Ruff — fast Python linter
# - ESLint — JS/TS linting
Section 03

Local Models for 16GB

With 16GB RAM, you want small-but-capable models. These are the best coding-focused picks that actually fit your machine.

!
Golden rule: Run ONE model at a time. Ollama auto-unloads after 5 minutes of inactivity. Don't try to keep multiple loaded.
Recommended Model Roster (Pick and Pull)
Model RAM Needed Best For Pull Command
Qwen 3 8B ~5GB Daily driver — fast general coding, questions, drafting ollama pull qwen3:8b
Qwen3 Coder 7B ~5GB Code generation, debugging, refactoring ollama pull qwen3-coder:7b
Gemma 3 4B ~3GB Ultra-fast classification, bug triage, simple tasks ollama pull gemma3:4b
DeepSeek-R1 14B ~9GB Reasoning, step-by-step problem solving, code review ollama pull deepseek-r1:14b
Gemma 3 12B ~8GB Classification, structured output, fallback general model ollama pull gemma3:12b
*
Start with just two: Pull qwen3:8b and qwen3-coder:7b on day one. Add others as you need them. Each model downloads once and stays on disk.
# Start Ollama server (runs in background)
ollama serve &

# Day 1: Pull your essentials
ollama pull qwen3:8b # ~5GB download
ollama pull qwen3-coder:7b # ~5GB download

# Later: Add reasoning model when you need it
ollama pull deepseek-r1:14b # ~9GB download

# Test it works
ollama run qwen3:8b "Write a Python function to check if a string is a palindrome"
Section 04

Claude Code Setup

Claude Code is your primary coding tool — it's an AI assistant that runs in your terminal, reads your project files, writes code, runs tests, and commits to git. Think of it as a senior developer in your terminal.

Get Your API Key
1

Create an Anthropic account

Go to console.anthropic.com and sign up. Add credit ($5–$20 to start).

2

Generate an API key

Go to API Keys in the console. Create a new key. Copy it — you'll only see it once.

3

Set it in your shell

Add this line to your ~/.zshrc and restart your terminal.

# Add to ~/.zshrc
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

# Reload
source ~/.zshrc
Basic Usage
# Navigate to your project
cd ~/Projects/my-app

# Start Claude Code
claude

# Or start with a specific task
claude "explain how the authentication flow works in this project"

# Choose model + effort level
claude --model sonnet # Fast + capable (default, cheapest)
claude --model opus # Most powerful (use for architecture, audits)
claude --model sonnet --effort high # Sonnet with deeper thinking
i
Cost tip: Use --model sonnet for most work (cheaper, fast). Switch to --model opus only for complex architecture decisions, debugging hard bugs, or reviewing code for security. Sonnet handles 80% of tasks.
What Claude Code Can Do

Read & Understand

  • Read your entire project structure
  • Understand how files relate to each other
  • Explain unfamiliar code
  • Search across files for patterns

Write & Edit

  • Write new files and functions
  • Edit existing code with precision
  • Refactor across multiple files
  • Add tests for your code

Run & Test

  • Execute shell commands
  • Run your test suite
  • Debug failing tests
  • Create git commits and branches
Section 05

Claude Models vs Local Ollama Equivalents

Claude is the gold standard, but local models are free, private, and work offline. Here's how they compare for coding tasks — honest assessment, not marketing.

Task Claude (Cloud) Best Ollama (16GB) How Close?
Code generation Sonnet 4.6 CLOUD Qwen3 Coder 7B LOCAL ~70% — handles simple-to-medium tasks well. Struggles with complex multi-file refactors.
Code review / audit Opus 4.6 CLOUD DeepSeek-R1 14B LOCAL ~55% — good chain-of-thought reasoning but misses subtle bugs. Use for first-pass only.
Quick questions Haiku 4.5 CLOUD Qwen 3 8B LOCAL ~80% — excellent for "how do I X", syntax lookups, quick explanations. Very close.
Bug classification Haiku 4.5 CLOUD Gemma 3 4B LOCAL ~75% — fast structured output. Great for triage: bug/feature/question sorting.
Architecture / planning Opus 4.6 CLOUD DeepSeek-R1 14B LOCAL ~45% — can outline ideas but lacks the depth for serious architecture. Use Claude.
Debugging complex issues Opus 4.6 CLOUD Qwen3 Coder 7B LOCAL ~50% — can help with straightforward bugs. For hairy issues, escalate to Claude.
Test writing Sonnet 4.6 CLOUD Qwen3 Coder 7B LOCAL ~65% — generates reasonable pytest scaffolding. May need manual cleanup on edge cases.
Explaining code Sonnet 4.6 CLOUD Qwen 3 8B LOCAL ~75% — clear explanations for most code. Misses nuance on very complex patterns.
*
The honest take: No 7B–14B local model matches Claude Sonnet or Opus for serious coding. They're useful for quick, free, offline tasks — not as replacements. Think of local models as your "quick lookup" tool and Claude as your "pair programmer".
When to Use Which

Use Claude Code (Cloud)

  • Writing new features or modules
  • Debugging tricky issues
  • Multi-file refactors
  • Architecture decisions
  • Security review / audit
  • Anything that goes to production

Use Ollama (Local)

  • "How do I do X in Python?" quick lookups
  • Explaining unfamiliar code snippets
  • Drafting commit messages or docs
  • Working offline (plane, train, cafe)
  • Working with private/sensitive data
  • Learning — experimenting without API costs
Section 06

Daily Workflow

A simple workflow to get you productive fast. No complex multi-pane setups — just what you need.

1

Open your terminal and project

Open iTerm2 (or Terminal). Navigate to your project: cd ~/Projects/my-app

2

Start coding with Claude

Type claude to start a session. Describe what you want to build, fix, or understand. Claude reads your files automatically.

3

Quick local questions with Ollama

For fast, free lookups: ollama run qwen3:8b "how do I parse JSON in Python" — no API cost, instant.

4

Test and commit

Ask Claude to run your tests: "run pytest and fix any failures". Then: "create a commit with a good message".

Task Routing Cheat Sheet
Situation Tool Command
Any coding task, no constraints Claude Code claude

Code generation offline Qwen3 Coder ollama run qwen3-coder:7b "..."
Reasoning through a problem DeepSeek-R1 ollama run deepseek-r1:14b "..."
Classify / triage Gemma 3 4B ollama run gemma3:4b "..."
Private/sensitive data Any Ollama model Data never leaves your machine
Section 07

Shell Aliases

Add these to your ~/.zshrc to make everything quicker. Copy the whole block.

# ─── AI CODING SETUP ───────────────────────────────────────────

# Anthropic API key
export ANTHROPIC_API_KEY="sk-ant-your-key-here"

# Ollama config
export OLLAMA_HOST="127.0.0.1:11434"
export OLLAMA_KEEP_ALIVE="5m"

# Python (prefer uv-managed environments)
export UV_PYTHON_PREFERENCE="managed"

# ─── QUICK ALIASES ──────────────────────────────────────────────

# Claude Code
alias cc='claude'
alias ccs='claude --model sonnet' # fast + capable
alias cco='claude --model opus' # most powerful

# Local models
alias llm='ollama run'
alias llm-fast='ollama run qwen3:8b' # daily driver
alias llm-code='ollama run qwen3-coder:7b' # code specialist
alias llm-reason='ollama run deepseek-r1:14b' # step-by-step reasoning
alias llm-classify='ollama run gemma3:4b' # ultra-fast classification

# Python environment
alias venv='python3 -m venv .venv && source .venv/bin/activate'
alias activate='source .venv/bin/activate'
After pasting: Run source ~/.zshrc to reload, or just open a new terminal tab.
