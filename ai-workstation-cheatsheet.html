Core Process

Step-by-Step

Click to expand. Each gate must pass before proceeding. Weight class determines which steps are mandatory.

01
Define the Problem
3 sentences: what exists, what's wrong, what success looks like. No tech names.
Manual ›
  • Write the 3-sentence problem statement.
  • Identify your user and their decision-making context.
  • Choose weight class: Prototype, Tool, or Product.
Gate Can you explain the problem without naming a technology? If not, think more.
02
Write Architecture Doc
ARCHITECTURE.md + CLAUDE.md + Mermaid diagrams. Review before any code.
AI-Assist ›
PLAN pane
  • Create ARCHITECTURE.md: system diagram, data flow, constraints.
  • Define pipeline stages. Name them. Fix the count. Best work: 3–6 stages.
  • Create CLAUDE.md with invariants, file map, hard rules.
  • If database: define migration strategy now. Add "never DROP directly" to CLAUDE.md.
  • Every doc gets a Last verified: date. If >10KB, split it.
Gate Has the architecture been reviewed before any code is written? "I'll review it later" = never.
03
Set Up the Safety Net
CI is step 3, not step 30. pytest + ruff + GitHub Actions before features.
Automate ›
IMPL pane
  • pytest configured, ruff in pre-commit, .env.example (never .env).
  • Write 3–5 smoke tests for the critical path before implementing it.
  • If UI exists: add a startup test now (Playwright or screenshot).
  • Set up GitHub Actions: lint + test on push. Use new-project.sh for scaffolding.
Gate Does CI run and pass on the empty project? If no — fix CI first. Zero features on a broken foundation.
04
Implement the Pipeline
Independent functions, typed I/O, caching from day one, scope check after each stage.
AI-Delegate ›
IMPL pane
  • Build each stage as an independent function with typed I/O.
  • SHA1-keyed caching for LLM responses, disk-persisted with TTL.
  • Graceful degradation: every try/except logs context and continues.
  • Extract hardcoded values into YAML config from the start.
  • Prototype: Single provider. Tool/Product: BaseLLMClient abstraction.
  • Scope check after each stage: "Am I still solving Step 1's problem?"
Gate 4a Does the pipeline produce correct output for the happy path?
Gate 4b Is scope still matching the problem statement from Step 1?
05
Cross-Model Audit
Opus reviews what Sonnet built. SECURITY_AUDIT.md with severity rankings.
Audit ›
AUDIT pane
  • Run full test suite (including frontend smoke tests). Fix failures before audit.
  • Security review: hardcoded secrets, path traversal, prompt injection, unescaped HTML.
  • Create SECURITY_AUDIT.md with severity-ranked findings.
  • Review CLAUDE.md — if >30% needs rewriting, revisit Step 2.
  • Use gate command in IMPL pane to run pre-ship checks.
Gate Are all HIGH/CRITICAL findings addressed? Fix before shipping. MEDIUM findings get a deadline.
06
Monitor & Alert
Monitor for silence, not just errors. Heartbeat checks + Telegram alerts.
Automate ›
IMPL pane
  • Structured logging with severity levels (DEBUG/INFO/WARNING/ERROR).
  • Cron pipelines: Heartbeat alert if pipeline didn't run or produced zero output.
  • User-facing apps: Sentry free tier or error webhook to Telegram.
Gate Does the pipeline alert when you simulate a failure? If no — monitoring is decorative.
07
Deploy with Automation
Dockerfile + CI/CD. Redeploy from scratch in under 5 minutes.
Automate ›
IMPL pane
  • Containerise: Dockerfile for backend.
  • Define deploy config: vercel.json, railway.json, or docker-compose.yml.
  • Automate: push to main → CI passes → deploy.
Gate Can you redeploy from scratch in under 5 minutes? If no — automate more.
08
Iterate & Triage
Stop rule, triage rule, done rule. Ship with documented limitations, not infinite polish.
Rules ›
  • Triage: HIGH/CRITICAL fix now. MEDIUM gets a deadline. LOW tracked, closed after 3 reviews.
  • Re-prompt rule: Include previous audit feedback to prevent duplicate suggestions.
  • Stop rule: Same feedback twice = stop retrying. Ship with documented limitation.
  • Done rule: All HIGH/CRITICAL fixed + happy path works + monitoring live + redeploy < 5 min.

Beginner-Friendly Guides

Media Generation

Step-by-step for images, audio, and video. Everything runs locally on your M5 Max — nothing leaves your machine.

IMG
Image Generation — Draw Things
Type a description, get an image. Free, local, runs on Apple GPU.
›
  • Open Draw Things from /Applications (or Spotlight: Cmd+Space → "Draw Things").
  • Type what you want in the prompt box. Be specific about style.
    Example: a calm mountain lake at sunset, digital art, soft lighting
  • Negative prompt (optional): type what to avoid in the second box.
    Example: blurry, text, watermark, low quality
  • Click Generate (or press Enter). Wait 15–50 seconds depending on model.
  • Save: Right-click the image → Save As. Default: 1024×1024.
Tips for better results Mention style: "watercolour", "photorealistic", "flat vector", "pencil sketch". Add lighting: "soft ambient", "golden hour", "dramatic shadows". More detail = better output. If something looks wrong, add to negative prompt rather than rewriting the whole thing.
Models available in Draw Things: Flux.1 [dev] (highest quality, ~30–50s) and SDXL Base (faster, ~15–25s). Download them inside the app on first use.
STT
Speech → Text — Whisper
Convert any audio/video file to text. One command, runs on GPU.
›
  • Transcribe any file: transcribe recording.wav — that's it.
  • Supported formats: .wav, .mp3, .m4a, .mp4, .webm — anything ffmpeg can read.
  • Output appears in your terminal. Runs locally — no internet needed.
  • Voice → Claude shortcut: voice-to-claude.sh — speak up to 30s, text auto-copies to clipboard. Paste with Cmd+V.
Under the hood Uses whisper-cli with the large-v3-turbo model (~1.5 GB, already downloaded at ~/.local/share/whisper-models/). Processes on Apple GPU. Excellent accuracy for English, good for most other languages.
TTS
Text → Speech — Kokoro
Make your Mac read text aloud with a natural voice. Local, no API key.
›
  • Step 1 — Activate the environment:
    source ~/kokoro-venv/bin/activate
  • Step 2 — Speak text aloud:
    python3 -c "from kokoro_onnx import Kokoro; import sounddevice as sd; t=Kokoro(); a,sr=t.create('Hello, this is a test.', voice='af_heart', speed=1.0); sd.play(a,sr); sd.wait()"
  • Step 3 — Deactivate when done: deactivate
Available voices & speed Female: af_heart (default), af_bella   Male: am_adam, am_michael   British: bf_emma, bm_george
Speed: 0.5 (slow) → 1.0 (normal) → 2.0 (fast). Change voice='...' and speed=... in the command.
AV
Audio/Video Processing — ffmpeg
Convert, trim, extract, resize. Copy-paste recipes — no editing knowledge needed.
›
  • Extract audio from video:
    ffmpeg -i video.mp4 -vn -acodec copy audio.aac
    Gets just the sound. -vn means "no video".
  • Convert any format:
    ffmpeg -i input.wav output.mp3
    Output format is auto-detected from the extension.
  • Trim a clip (start at 1 min, keep 30 sec):
    ffmpeg -i input.mp4 -ss 00:01:00 -t 00:00:30 -c copy output.mp4
    -ss = start time, -t = duration.
  • Record system audio (30 seconds):
    ffmpeg -f avfoundation -i ":0" -t 30 recording.wav
  • Resize video to 720p:
    ffmpeg -i input.mp4 -vf scale=1280:720 output.mp4
  • Create GIF from first 5 seconds:
    ffmpeg -i input.mp4 -t 5 -vf "fps=10,scale=480:-1" output.gif
The pattern Always: ffmpeg -i [input] [options] [output]. When in doubt, ask Claude: "ffmpeg command to [what I want]". It will give you the exact command.
REC
Screenshots & Screen Recording
Built into macOS. No extra tools needed.
›
  • Full screenshot: Cmd+Shift+3
  • Area screenshot: Cmd+Shift+4 → drag to select
  • Window screenshot: Cmd+Shift+4 then Space → click window
  • Recording toolbar: Cmd+Shift+5 → choose record area or full screen
  • Copy screenshot to clipboard (instead of file): add Ctrl to any shortcut above
  • All files save to ~/Desktop by default.
VID
Video Generation — Cloud Only (for now)
No local video gen yet. These cloud tools are the current best options.
›
  • Sora (OpenAI): Text → video. Best quality. Needs ChatGPT Plus sub.
  • Runway Gen-3: Text or image → video. Good for short clips. Free tier available.
  • Kling (Kuaishou): Free tier, good quality, longer clips (up to 10s).
  • Workflow: Generate with Draw Things (image) → animate with Runway/Kling (video).
  • Use /prompt-master in Claude to write optimised prompts for any of these tools.
Local video gen is not practical yet on consumer hardware. When it is, Stability Matrix (ComfyUI) will likely be the first option. Install with: brew install --cask stability-matrix.

Mental Model

3 Key Principles

Pipeline Discipline

Fix the stages. Name them. Never add stages mid-flight. 3–6 stages with typed boundaries. Predictability beats flexibility.

Trust but Verify

LLMs are primary, never sole. Every AI output has a verification layer — numbers checked, schemas validated, different model reviews.

Degrade Gracefully

Every feature in try/except with context logging. LLM down? Pipeline still produces. Cache cold? Rebuilds silently. Nothing crashes delivery.

Iteration Rules

Bug Triage

HIGH / CRITICAL

Fix now, before the next feature. No exceptions. These block shipping.

MEDIUM

Create tracking issue with a deadline. If deadline passes: fix it or downgrade with rationale.

LOW

Track it. If still LOW after 3 reviews, close as "accepted limitation."

Stop rule: Same audit feedback twice = stop retrying. Ship with documented limitation. Endless retries need human thinking, not more AI.
